## 1. Login — Automated Test Scenarios

### TC-LG-001 — Успешный вход (standard_user)

**Тип:** Functional, Smoke

**Предусловия:**
- Приложение запущено
- Пользователь не авторизован
- Отображается экран Login

**Шаги:**
1. Ввести username: standard_user
2. Ввести password: secret_sauce
3. Нажать кнопку LOGIN

**Ожидаемый результат:**
- Отображается экран PRODUCTS
- Видно минимум 1 товар
- Корзина показывает 0

**Постусловия:**
- Пользователь авторизован и сессия активна

### TC-LG-002 — Вход с locked_out_user

**Тип:** Negative

**Предусловия:** Экран Login

**Шаги:**
1. Ввести username: locked_out_user
2. Ввести password: secret_sauce
3. Нажать LOGIN

**Ожидаемый результат:**
- Ошибка: "Sorry, this user has been locked out."
- Пользователь остаётся на Login

**Постусловия:**
- Сессия не создаётся

### TC-LG-003 — Неверный пароль

**Тип:** Negative

**Предусловия:** Экран Login

**Шаги:**
1. Ввести username: standard_user
2. Ввести неверный пароль invalid123
3. Нажать LOGIN

**Ожидаемый результат:**
- Ошибка: "Username and password do not match any user in this service"

**Постусловия:**
- Пользователь не авторизован

### TC-LG-004 — Пустое имя пользователя

**Тип:** Validation

**Предусловия:** Экран Login

**Шаги:**
1. Оставить username пустым
2. Ввести password: secret_sauce
3. Нажать LOGIN

**Ожидаемый результат:**
- Ошибка: "Username is required"

**Постусловия:**
- Пользователь остаётся на Login

## 2. Products — Automated Test Scenarios

### TC-PL-001 — Отображение экрана продуктов

**Тип:** Smoke, Functional

**Предусловия:** Пользователь залогинен → отображается экран PRODUCTS

**Шаги:**
1. Проверить заголовок "PRODUCTS"
2. Проверить наличие минимум 1 товар

**Ожидаемый результат:**
- Видны товары
- Каждая карточка содержит картинку, имя, цену, кнопку ADD TO CART

**Постусловия:**
- Страница доступна для взаимодействия

### TC-PL-002 — Добавление товара в корзину

- Кнопка меняется на REMOVE
- Цвет кнопки меняется на красный
- Badge в корзине = 1

**Постусловия:**
- В корзине 1 товар
